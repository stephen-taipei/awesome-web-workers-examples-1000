# 配額限制 (Quota Limit)

此範例展示如何在 Web Worker 中實作資源配額限制機制。

## 功能

- 設定操作次數的配額。
- 每次執行操作時，Worker 會檢查是否超過配額。
- 若配額足夠，執行操作並更新剩餘配額；若配額不足，拒絕操作並通知主執行緒。
- 提供重置功能。

## 使用方式

1. 輸入配額數量並點擊 "Set Quota" (預設為 50)。
2. 點擊 "Perform Operation" 執行模擬操作。
3. 觀察操作次數與剩餘配額的變化。
4. 當配額用盡時，操作按鈕將被停用，且顯示錯誤訊息。
5. 點擊 "Reset" 重置計數器。

## 技術重點

- **狀態管理**: Worker 內部維護配額與已使用量的狀態。
- **條件判斷**: 在執行請求前檢查配額限制。
- **雙向通訊**: 主執行緒發送指令，Worker 回報結果或錯誤。
