Web Worker Example #228 - Priority Queue for Worker Messages

About Message Priority

This example demonstrates a priority-based message queue system for Web Workers. Messages are processed based on their priority level, ensuring important tasks are handled first.

Priority Queue Implementation:

Priority Levels:
  CRITICAL (4) - Immediate processing
  HIGH (3)     - Process before normal tasks
  NORMAL (2)   - Default priority
  LOW (1)      - Background tasks

Processing Order:
  1. Extract highest priority message
  2. Process message in Worker
  3. Return result with timing info
  4. Continue with next highest priority

Binary Heap Operations:
  Insert:  O(log n)
  Extract: O(log n)
  Peek:    O(1)
