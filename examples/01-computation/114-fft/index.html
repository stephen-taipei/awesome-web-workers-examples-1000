About Fast Fourier Transform

The FFT is an efficient algorithm for computing the DFT in O(N log N) time instead of O(N²), making large-scale frequency analysis practical.

Cooley-Tukey Algorithm

X[k] = X_even[k] + W_N^k · X_odd[k]
where W_N = e^(-2πi/N) is the twiddle factor

Divide and Conquer

Split
x[n] → x_even[n], x_odd[n]
Recurse
FFT(N) = 2×FFT(N/2) + O(N)
Combine
Butterfly: X[k], X[k+N/2]

Complexity Comparison

  • DFT: O(N²) - 1M samples ≈ 10¹² operations
  • FFT: O(N log N) - 1M samples ≈ 20M operations
  • Speedup: ~50,000× for N = 10⁶

Requirements

  • N must be a power of 2 (radix-2 FFT)
  • Bit-reversal permutation for in-place computation
  • Twiddle factors can be precomputed
