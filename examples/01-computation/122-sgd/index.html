About Stochastic Gradient Descent

SGD updates parameters using a subset (mini-batch) of training data, providing a balance between the noise of pure stochastic updates and the computational cost of full batch gradient descent.

Update Rule

θ_{t+1} = θ_t - α · ∇L_B(θ_t)
B = mini-batch, ∇L_B = average gradient over batch

Momentum Update

Velocity Update
v_t = βv_{t-1} + ∇L_B(θ_t)
Parameter Update
θ_{t+1} = θ_t - α · v_t

Batch Size Trade-offs

  • Small batch (1-16): Noisy updates, escapes local minima, high variance
  • Medium batch (32-128): Good balance, efficient GPU utilization
  • Large batch: Stable but may converge to sharp minima
  • Full batch: Deterministic, slow per-epoch, true gradient

Advantages of SGD

  • Scalability: Works with huge datasets that don't fit in memory
  • Regularization: Noise helps escape sharp minima
  • Online learning: Can update with streaming data
  • Faster iterations: Many small updates vs few large ones
