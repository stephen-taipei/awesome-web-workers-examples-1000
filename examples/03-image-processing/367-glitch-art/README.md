# Glitch Art

此示例展示了如何使用 Web Worker 模擬故障藝術（Glitch Art）效果。

## 技術原理

故障藝術是透過模擬數位訊號錯誤或資料損壞來創造的視覺風格。此示例組合了多種處理技術：

1.  **行位移 (Row Shifting)**：
    *   隨機選擇圖像的水平區塊。
    *   將該區塊內的像素向左或向右平移隨機距離。
    *   超出的部分會循環（Wrap）到另一側，模擬訊號同步錯誤。

2.  **RGB 通道分離 (RGB Split)**：
    *   隨機選擇一個顏色通道（R、G 或 B）。
    *   將該通道的數據整體平移，造成顏色錯位。

3.  **區塊損壞 (Block Corruption)**：
    *   隨機選擇圖像中的矩形區域。
    *   對該區域應用不同的破壞效果：
        *   **反色 (Invert)**：反轉像素顏色。
        *   **噪點 (Noise)**：填充隨機灰階噪點。
        *   **像素排序 (Pixel Sorting)**：按亮度對區塊內的每一行像素進行排序，產生拉伸效果。

## 參數說明

*   **強度 (Intensity)**：控制故障的嚴重程度（位移距離、損壞區塊數量等）。
*   **種子 (Seed)**：控制隨機數生成器的初始狀態。相同的種子會產生相同的故障圖案，這對於動畫或調整特定效果很有用。

## 實作細節

*   使用自定義的偽隨機數生成器（基於 `Math.sin`），以確保在 Worker 中可重現相同的隨機序列。
*   大量使用 `Uint8ClampedArray` 進行像素操作。
*   像素排序操作涉及陣列排序，計算成本較高，僅在局部區塊應用以保持效能。
