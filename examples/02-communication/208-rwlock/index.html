Read-Write Lock (RWLock)

Web Worker Example #208 - SharedArrayBuffer Synchronization

About RWLock

A Read-Write Lock allows multiple concurrent readers OR a single exclusive writer. This is efficient for read-heavy workloads where data is read frequently but modified rarely.

RWLock States:

Lock State Layout (SharedArrayBuffer):
┌─────────────────────────────────────────┐
│ state[0] = lock state                   │
│   0: unlocked                           │
│   > 0: number of active readers         │
│   -1: writer holding lock               │
├─────────────────────────────────────────┤
│ state[1] = waiting writers count        │
├─────────────────────────────────────────┤
│ state[2] = shared data value            │
└─────────────────────────────────────────┘

Read Lock:  Wait until no writer, increment reader count
Write Lock: Wait until no readers/writers, set to -1
Unlock:     Decrement count or release writer lock

Uses Atomics.wait() and Atomics.notify() for blocking
