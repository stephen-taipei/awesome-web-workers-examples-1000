Web Worker Example #209 - Atomics.wait/notify Synchronization

About Condition Variables

A Condition Variable allows threads to wait for a specific condition to become true. Combined with a mutex, it enables efficient producer-consumer patterns where consumers wait for data and producers signal when data is available.

Condition Variable Pattern:

Producer-Consumer with Condition Variable:

Producer:
  1. Acquire mutex lock
  2. Produce data into buffer
  3. Signal condition (wake consumers)
  4. Release mutex lock

Consumer:
  1. Acquire mutex lock
  2. While (buffer is empty):
       Wait on condition (releases lock, sleeps)
  3. Consume data from buffer
  4. Release mutex lock

SharedArrayBuffer Layout:
┌──────────────────────────────────────────┐
│ [0] mutex        - 0: unlocked, 1: locked │
│ [1] condition    - wait/notify target     │
│ [2] buffer_count - items in buffer        │
│ [3..N] buffer    - circular data buffer   │
└──────────────────────────────────────────┘

Uses Atomics.wait() to block until signaled
Uses Atomics.notify() to wake waiting threads
