關於此範例

  • 功能：實作訊息重傳機制，失敗時自動重試直到成功或達到最大重試次數
  • 退避策略：支援固定延遲、線性退避、指數退避三種策略
  • Worker 類型：Dedicated Worker (專屬 Worker)
  • 重傳佇列：使用佇列管理待重傳的訊息，確保順序處理
  • 通訊模式：使用 postMessage 進行雙向通訊，含訊息 ID 追蹤
