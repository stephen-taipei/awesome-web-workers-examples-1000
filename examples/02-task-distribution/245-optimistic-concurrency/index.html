About Optimistic Concurrency

Optimistic Concurrency Control (OCC) is a concurrency control method that assumes conflicts are rare. Transactions proceed without locking, and conflicts are detected at commit time. If a conflict is found, the transaction is rolled back and retried.

OCC Transaction Phases:

1. Read Phase:
   - Read data into private workspace
   - Track read set and write set
   - No locks acquired

2. Validation Phase:
   - Check for conflicts with committed transactions
   - Verify read set hasn't changed
   - Ensure serializability

3. Write Phase:
   - If validation succeeds, commit changes
   - If validation fails, abort and retry

Conflict Detection:
   - Write-Write: Two transactions modify same data
   - Read-Write: Data read was modified by another

Advantages:
   - High concurrency when conflicts are rare
   - No deadlocks possible
   - Better throughput for read-heavy workloads
