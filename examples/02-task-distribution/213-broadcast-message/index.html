Broadcast Message

#213 通訊協調 中級

功能說明

此範例展示如何使用 BroadcastChannel API 實現一對多的訊息廣播，讓主執行緒和多個 Worker 之間能夠同步通訊。

  • BroadcastChannel 允許同源的不同視窗、Tab 或 Worker 通訊
  • 訊息會自動廣播給所有訂閱同一頻道的接收者
  • 適合需要同步狀態或發送全域通知的場景
  • 比逐一發送 postMessage 更簡潔高效

Worker 管理

尚未建立任何 Worker

通訊記錄

尚無通訊記錄

關於此範例

  • 通訊模式：一對多廣播 (One-to-Many Broadcast)
  • 核心技術：使用 BroadcastChannel API
  • 頻道概念：同一頻道名稱的所有端點都會收到訊息
  • 適用場景：狀態同步、全域通知、配置更新
  • 注意事項：發送者不會收到自己發送的訊息
