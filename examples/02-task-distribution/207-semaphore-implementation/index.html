Web Worker Example #207 - Counting Semaphore for Resource Limiting

About Semaphores

A Semaphore is a synchronization primitive that maintains a count. Unlike a mutex (binary semaphore), a counting semaphore allows multiple threads to access a limited resource pool simultaneously.

Semaphore Implementation:

// Semaphore with count N allows N concurrent accesses

class Semaphore {
    constructor(sharedArray, index, maxCount) {
        this.array = sharedArray;
        this.index = index;
        Atomics.store(this.array, this.index, maxCount);
    }

    acquire() {
        while (true) {
            const current = Atomics.load(this.array, this.index);
            if (current > 0) {
                // Try to decrement
                const old = Atomics.compareExchange(
                    this.array, this.index, current, current - 1
                );
                if (old === current) return; // Got a permit
            }
            // Wait for a permit to become available
            Atomics.wait(this.array, this.index, 0);
        }
    }

    release() {
        Atomics.add(this.array, this.index, 1);
        Atomics.notify(this.array, this.index, 1);
    }
}

Use cases:
- Connection pool (limit concurrent connections)
- Rate limiting (limit concurrent requests)
- Resource pool (limit concurrent file handles)
