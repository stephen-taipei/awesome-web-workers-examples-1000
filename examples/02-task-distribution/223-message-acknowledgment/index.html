ACK 機制示意圖

主執行緒                                Worker
   │                                      │
   │──── MSG #1 (seq=1) ────────────────→│
   │                                      │ 處理訊息
   │←─────── ACK #1 ──────────────────────│
   │                                      │
   │──── MSG #2 (seq=2) ────────────────→│
   │         ✕ (ACK 丟失)                 │
   │                                      │
   │    [超時，重傳]                       │
   │                                      │
   │──── MSG #2 (seq=2, retry=1) ───────→│
   │←─────── ACK #2 ──────────────────────│
   │                                      │

關於此範例

  • 序號追蹤：每則訊息分配唯一序號，用於識別和確認
  • ACK 機制：Worker 處理完訊息後回傳確認 (ACK)，發送端據此判斷成功
  • 超時重傳：超過設定時間未收到 ACK，自動重傳訊息
  • Worker 類型：Dedicated Worker (專屬 Worker)
  • 應用場景：可靠訊息傳遞、命令執行確認、資料同步
