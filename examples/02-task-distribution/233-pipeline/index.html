About Pipeline Pattern

Pipeline 模式將處理流程分解為多個連續的階段（Stage），每個階段由獨立的 Worker 處理。資料像流水線一樣依序經過各階段，實現串流處理。這種模式特別適用於需要多步驟處理的場景，可以同時處理多個項目。

Pipeline 執行流程：

Input Stream → [Stage 1] → [Stage 2] → [Stage 3] → Output

┌─────────────────────────────────────────────────────────┐
│   Time →                                                │
│   ┌───────┐                                             │
│   │Item 1 │→ Stage1 → Stage2 → Stage3 → Output         │
│   └───────┘                                             │
│       ┌───────┐                                         │
│       │Item 2 │→ Stage1 → Stage2 → Stage3 → Output     │
│       └───────┘                                         │
│           ┌───────┐                                     │
│           │Item 3 │→ Stage1 → Stage2 → Stage3 → Output │
│           └───────┘                                     │
└─────────────────────────────────────────────────────────┘

特點：
• 各階段並行執行不同項目
• 持續吞吐量穩定
• 適合資料串流處理
• 延遲 = 單項目通過所有階段的時間
• 吞吐量 = 最慢階段的處理速度
