Master-Worker Pattern

Web Worker Example #235 - 主從模式

About Master-Worker Pattern

Master-Worker 模式由一個主控節點（Master）動態分配任務給多個工作節點（Worker）。Master 負責任務排程、負載平衡和結果收集。這種模式具有高度可擴展性，適用於任務數量不定或處理時間差異大的場景。

Master-Worker 執行流程：

           ┌─────────────────────────────────────┐
           │            MASTER                   │
           │  ┌───────────────────────────────┐  │
           │  │   Task Queue (待處理任務)      │  │
           │  │   [T1][T2][T3][T4][T5]...     │  │
           │  └───────────────────────────────┘  │
           │            │ 分配任務               │
           └────────────┼────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐
   │ Worker1 │    │ Worker2 │    │ Worker3 │
   │  處理T1  │    │  處理T2  │    │  處理T3  │
   └────┬────┘    └────┬────┘    └────┬────┘
        │               │               │
        └───────────────┼───────────────┘
                        │ 回報結果
                        ▼
           ┌─────────────────────────────────────┐
           │            MASTER                   │
           │  • 收集結果                         │
           │  • 分配下一個任務給空閒 Worker      │
           │  • 負載平衡                         │
           └─────────────────────────────────────┘

特點：
• 動態任務分配（Work Stealing）
• 自動負載平衡
• 容錯處理（Worker 失敗可重新分配）
• 高度可擴展
