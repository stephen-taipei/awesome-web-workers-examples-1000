How it works

  • UUID v4: Uses random numbers for all significant bits (122 bits of randomness).
  • UUID v1: Based on timestamp and node ID (MAC address simulated).
  • The Web Worker generates UUIDs in batches without blocking the main thread.
  • Multiple output formats supported for different use cases.
