# 隨機分配 (Random) 負載均衡

本範例展示如何實作隨機分配 (Random) 負載均衡策略。

## 原理

隨機分配演算法非常簡單，對於每一個新任務，從 Worker 池中隨機選擇一個 Worker 來處理。
`worker = workers[Math.floor(Math.random() * workers.length)]`

## 特點

*   **優點**：實作極其簡單，無需維護狀態（如輪詢索引或連接數）。在大量請求下，根據機率分佈，負載會趨於均衡。
*   **缺點**：隨機性可能導致短時間內的負載不均（例如運氣不好，所有任務都分配給了同一個 Worker）。不保證絕對的順序或公平。

## 實作細節

*   使用 `Math.random()` 生成隨機索引。
*   選擇對應的 Worker 進行任務派發。
*   透過視覺化介面可以觀察到，任務數量的成長雖然大致平均，但會有隨機的波動。
