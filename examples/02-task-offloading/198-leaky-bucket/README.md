# Web Worker 漏桶演算法 (Leaky Bucket)

此示例演示了 **漏桶演算法 (Leaky Bucket)**。這是一種流量整形 (Traffic Shaping) 演算法。

## 原理

*   **桶 (Bucket)**: 具有固定容量。
*   **進水 (Inflow)**: 請求進入桶中（如同水滴入桶）。如果請求到達時桶已滿，則請求被丟棄（溢出）。
*   **漏水 (Outflow)**: 水以恆定的速率從桶底漏出（請求被處理）。這保證了輸出的流量是平滑且恆定的，無論輸入流量有多麼突發。

## 功能

1.  **模擬請求**: 點擊按鈕向桶中加水。
2.  **恆定流出**: 觀察桶中的水位以恆定速率下降。
3.  **溢出處理**: 當水位達到容量上限時，新的請求將被拒絕。
4.  **動態配置**: 可調整桶容量和漏水速率。

## 使用方式

1.  設定「桶容量」和「漏水速率」。
2.  點擊「發送請求」向桶中加水。
3.  觀察水位上升和下方的漏水動畫。
4.  嘗試快速點擊「大量請求」，使水位達到頂端，觀察日誌中的「溢出丟棄」事件。
