About Peer-to-Peer Pattern

The Peer-to-Peer (P2P) pattern enables direct communication between Workers without a central coordinator. Each Worker can send messages to any other Worker, creating a decentralized mesh network. This pattern is ideal for distributed algorithms like gossip protocols, distributed consensus, and collaborative computations.

P2P Architecture:

Traditional (Hub-and-Spoke):          Peer-to-Peer (Mesh):

     Main Thread                      Worker A ←→ Worker B
         ↓↑                               ↕   ╲   ╱   ↕
    ┌────┼────┐                           ↕    ╲ ╱    ↕
    ↓    ↓    ↓                           ↕     ╳     ↕
   W1   W2   W3                           ↕    ╱ ╲    ↕
                                      Worker C ←→ Worker D

Key Concepts:
• MessageChannel - Creates port pairs for direct communication
• transferable - Ports are transferred, not copied
• Gossip Protocol - Information spreads through peer messages
• Consensus - Peers agree on shared state without central authority

Use Cases:
• Distributed sorting/merging
• Collaborative filtering
• Decentralized computation
• Fault-tolerant systems
