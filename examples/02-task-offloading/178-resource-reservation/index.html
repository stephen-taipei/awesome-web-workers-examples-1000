關於資源預留

  • 資源隔離：將部分 Worker 劃分為「預留池」，專門處理高優先級任務。
  • 服務品質保證 (QoS)：即使普通任務大量湧入佔滿了共用池，預留池仍能保持空閒，確保關鍵任務（如 UI 互動、即時回應）能被立即處理。
  • 調度策略：
    • 普通任務：只能使用共用資源池。若共用池滿，則進入等待佇列。
    • 關鍵任務：優先使用預留池，若預留池滿，也可借用共用池（如果共用池有空閒）。
