About Task Parallelism

Task Parallelism executes different, independent tasks concurrently across multiple workers. Unlike data parallelism where the same operation runs on different data, task parallelism handles heterogeneous tasks - each worker performs a distinct computation type.

Task Parallel Model:

Task Parallelism vs Data Parallelism:

Data Parallel:   [Worker1: f(data1)] [Worker2: f(data2)] [Worker3: f(data3)]
                 Same function f(), different data chunks

Task Parallel:   [Worker1: Task_A()] [Worker2: Task_B()] [Worker3: Task_C()]
                 Different functions, independent tasks

Benefits:
- Maximum resource utilization
- Reduced total execution time
- Independent failure handling
- Natural fit for heterogeneous workloads

Total Time = max(Task_A_time, Task_B_time, Task_C_time)
Speedup = (T_A + T_B + T_C) / max(T_A, T_B, T_C)
