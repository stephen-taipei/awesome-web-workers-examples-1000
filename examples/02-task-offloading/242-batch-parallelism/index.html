About Batch Parallelism

Batch Parallelism groups multiple small tasks into batches for efficient parallel processing. Instead of processing items one-by-one, batching reduces scheduling overhead, communication costs, and improves throughput for large numbers of small tasks.

Batch Processing Model:

Individual Processing (N items, K workers):
  Overhead per item: O_comm + O_schedule
  Total overhead: N × (O_comm + O_schedule)

Batch Processing (N items, B batches, K workers):
  Items per batch: N / B
  Total overhead: B × (O_comm + O_schedule)
  Savings: (N - B) × overhead_per_item

Optimal Batch Size:
  Too small → High overhead (like individual)
  Too large → Poor load balancing
  Optimal ≈ N / (K × 2-4)

Throughput = Items / Time
Efficiency = Actual_throughput / Theoretical_max
