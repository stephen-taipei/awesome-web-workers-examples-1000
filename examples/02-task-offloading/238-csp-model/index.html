About CSP Model

Communicating Sequential Processes (CSP) is a formal language for describing patterns of interaction in concurrent systems. Unlike shared memory, CSP processes communicate through channels - typed conduits through which values are sent and received synchronously.

CSP Channel Operations:

Channel Types:
┌─────────────────────────────────────────────┐
│  Unbuffered Channel (Synchronous)           │
│  ┌─────┐    send()    ┌─────┐              │
│  │ P1  │ ──────────→  │ Ch  │  ← blocked   │
│  └─────┘   blocked    └─────┘    until     │
│            until       │  ↓      recv()    │
│            recv()      ▼                    │
│                     ┌─────┐                 │
│                     │ P2  │  recv()        │
│                     └─────┘                 │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Buffered Channel (Asynchronous)            │
│  ┌─────┐            ┌───────────┐          │
│  │ P1  │ ─send()─→  │ [v1,v2,v3]│ buffer   │
│  └─────┘            └───────────┘          │
│   non-blocking          │                   │
│   until full            ▼                   │
│                     ┌─────┐                 │
│                     │ P2  │  recv()        │
│                     └─────┘  non-blocking  │
│                              until empty   │
└─────────────────────────────────────────────┘

Patterns:
• Pipeline: A → ch1 → B → ch2 → C
• Fan-out:  A → ch → [B1, B2, B3]
• Fan-in:   [A1, A2, A3] → ch → B
• Select:   wait for first of multiple channels
