# Web Worker 令牌桶演算法 (Token Bucket)

此示例演示了 **令牌桶演算法 (Token Bucket)**。這是一種常見的限流演算法，特別適合允許一定程度突發流量的場景。

## 原理

*   **令牌生成**: 系統以恆定的速率生成令牌並放入桶中。
*   **桶容量**: 桶有最大容量，多餘的令牌會被丟棄（不會無限累積）。
*   **請求處理**: 每個請求需要消耗一定數量的令牌。如果桶中有足夠的令牌，請求被允許並消耗令牌；否則，請求被拒絕。
*   **突發流量**: 由於桶中可以積累令牌，系統可以在短時間內處理大量請求（直到令牌耗盡），這稱為「突發允許」。

## 與漏桶演算法的區別

*   **漏桶 (Leaky Bucket)**: 強制輸出速率恆定，平滑流量。
*   **令牌桶 (Token Bucket)**: 允許突發流量，只要平均速率不超過限制。

## 使用方式

1.  設定「桶容量」和「生成速率」。
2.  觀察令牌自動生成並填滿桶。
3.  點擊「發送請求」消耗一個令牌。
4.  點擊「突發請求」一次消耗 5 個令牌，觀察當桶滿時可以立即處理，但隨後需要等待令牌生成。
