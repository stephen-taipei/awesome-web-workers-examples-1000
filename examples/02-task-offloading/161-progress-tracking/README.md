# 161. 進度追蹤 (Progress Tracking)

本範例展示如何在長任務執行過程中，從 Web Worker 回報進度給主執行緒，並更新 UI 進度條。

## 功能說明

- 啟動一個模擬大量運算（100 萬次浮點運算）的任務。
- Worker 計算過程中，每完成 1% 進度就發送 `postMessage` 回報。
- 主執行緒接收進度資訊並即時更新 HTML 進度條。

## 技術重點

1.  **定期回報**: 為了避免通訊過於頻繁導致效能下降（postMessage 序列化與事件處理也有開銷），通常不應對每個小步驟都回報，而是設定適當的回報間隔（例如每 1% 或每 100ms）。
2.  **UI 更新**: 主執行緒收到訊息後更新 DOM。由於 Web Worker 是並行的，UI 依然保持流暢響應。

## 使用方式

1. 點擊「開始長任務」。
2. 觀察進度條平滑移動直到 100%。
