# 162. 結果快取 (Result Caching)

本範例展示如何在前端使用 LRU (Least Recently Used) 快取策略來管理 Web Worker 的計算結果。

## 功能說明

- 請求計算費波那契數列 (Fibonacci) 的第 N 項。
- 主執行緒維護一個容量為 5 的 LRU Cache。
- **Cache Hit**: 如果 N 的結果已在快取中，直接返回，不啟動 Worker (回應時間接近 0)。
- **Cache Miss**: 如果不在快取中，啟動 Worker 進行計算，並將結果存入快取。
- 當快取滿時，會自動移除最久未使用的項目。

## 技術重點

1.  **LRU Cache**: 使用 JavaScript 的 `Map` 物件特性（保持插入順序）來實作高效的 LRU 快取。
2.  **效能優化**: 對於重複的昂貴計算請求，透過快取機制可以完全消除計算成本。

## 使用方式

1. 輸入一個數字 (建議 35-45 之間，計算時間較明顯)。
2. 點擊「計算」，觀察是「快取未命中」並等待計算結果。
3. 再次點擊「計算」相同的數字，觀察是「快取命中」且立即顯示結果。
4. 嘗試輸入不同的數字直到快取滿 (容量 5)，觀察舊項目被移除。
