# 地理位置負載均衡 (Location Based Balancing)

此示例演示了基於地理位置 (Geo-Location) 的負載均衡策略。系統會根據發出請求的座標，自動將其導向最近的資料中心，以實現最低的網路延遲。

## 功能

1.  **世界地圖模擬**：
    *   使用 Canvas 繪製一個簡化的地圖區域。
    *   標記了全球主要區域的模擬資料中心 (Data Center, DC)，如美東、美西、歐洲、亞洲等。
2.  **互動式請求**：
    *   **點擊地圖**：使用者可以點擊地圖上的任意位置來模擬發出一個請求。
    *   系統會計算該位置到所有資料中心的歐幾里得距離，並選擇最近的一個。
    *   視覺化顯示請求到目標資料中心的連線，並模擬延遲。
3.  **自動模式**：可以開啟自動模式，隨機產生全球各地的請求，觀察整體的流量分布。
4.  **統計分析**：即時統計每個資料中心處理的請求數量和佔比。

## 技術重點

*   **距離計算**：Worker 執行緒中進行距離計算 (在此示例中為簡單的歐式距離)，在實際應用中可能涉及更複雜的 GeoIP 資料庫查詢或 Great-circle distance 計算。
*   **延遲模擬**：根據距離動態計算模擬延遲。
*   **Canvas 視覺化**：繪製資料中心、請求點和連線。

## 使用方式

1.  開啟 `index.html`。
2.  在地圖空白處點擊，模擬使用者請求。觀察日誌中顯示的「路由目標」和「延遲」。
3.  勾選「自動產生請求」，觀察大量隨機請求下的流量分布情況。
