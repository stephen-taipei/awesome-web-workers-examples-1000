# 降級策略 (Degradation Strategy)

此範例展示如何在 Web Workers 應用中實作服務降級 (Service Degradation) 策略。當系統負載過高或資源不足時，自動切換至低消耗、低品質的處理模式，以維持系統的基本可用性。

## 核心概念

*   **服務分級**: 定義不同的服務品質級別。
    *   **正常模式**: 提供完整功能、高品質結果，但消耗較多資源 (CPU/時間)。
    *   **降級模式**: 提供簡化功能、可接受的結果，但消耗極少資源。
*   **觸發機制**: 根據系統當前負載 (Load) 決定使用哪種模式。在此範例中，透過手動拖動滑桿模擬負載變化。

## 範例說明

本範例以圖像處理為場景：
*   **高品質模式 (負載 < 80%)**: 模擬複雜的圖像演算法 (耗時約 500ms)，佔用較多 CPU 時間。
*   **降級模式 (負載 ≥ 80%)**: 使用極簡的圖像演算法 (耗時 < 10ms)，快速返回結果，避免 Worker 堆積過多任務。

## 使用方法

1.  調整「負載模擬」滑桿。
2.  觀察「服務狀態」指示燈。
    *   綠色 (正常): 負載低於 80%。
    *   紅色 (降級): 負載高於或等於 80%。
3.  點擊「執行處理」按鈕。
4.  觀察「處理結果」下方的耗時與模式說明。在降級模式下，處理速度會顯著加快，但圖像效果可能較為簡單（本例中兩者視覺差異不大，重點在於模擬的計算延遲差異）。
