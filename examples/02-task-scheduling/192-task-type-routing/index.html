關於此範例

  • 核心概念：根據任務特性將任務路由到專門的 Worker。
  • Worker 類型：
    • CPU Worker: 處理計算密集型任務 (如數學運算)。
    • IO Worker: 處理 IO 密集型任務 (如網路請求模擬)。
  • 優點：
    • 避免長計算阻塞 IO 操作。
    • 可以針對不同類型的任務優化 Worker 環境或資源配置。
