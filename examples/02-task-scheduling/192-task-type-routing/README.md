# 任務類型分流 (Task Type Routing)

此範例展示如何根據任務的類型將任務分發到不同的 Web Worker。這是一種常見的負載均衡策略，特別適用於任務性質差異很大的場景（例如計算密集型 vs IO 密集型）。

## 功能

*   **CPU Worker**: 專門處理計算密集型任務（遞迴計算費波那契數）。
*   **IO Worker**: 專門處理 IO 密集型任務（模擬網路請求延遲）。
*   **任務分流**: 主執行緒根據使用者選擇的任務類型，將任務 `postMessage` 到相應的 Worker。
*   **狀態監控**: 即時顯示各個 Worker 的狀態（閒置/工作中）與佇列長度（這裡簡化為待處理數）。

## 為什麼要這樣做？

*   **避免阻塞**: 計算密集型任務如果放在主執行緒會導致 UI 凍結。如果放在處理 IO 的 Worker 中，可能會延遲 IO 事件的處理。
*   **資源優化**: 可以為不同類型的 Worker 分配不同的資源或數量。例如，IO 密集型任務通常不佔用 CPU，可以開啟更多 Worker；而 CPU 密集型 Worker 的數量通常受限於物理核心數。

## 使用方法

1.  選擇任務類型 (CPU 或 IO)。
2.  選擇任務複雜度。
3.  點擊「提交任務」或「隨機提交 10 個任務」。
4.  觀察下方的系統狀態與任務日誌。
